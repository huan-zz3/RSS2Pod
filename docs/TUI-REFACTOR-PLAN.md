# RSS2Pod CLI to TUI Refactoring Plan

**生成时间:** 2026-03-13
**状态:** 待执行
**预计工期:** 4 周

## 执行摘要

**目标:** 将 RSS2Pod 的 24+ 命令 CLI 重构为交互式文本用户界面 (TUI)，采用菜单驱动导航，同时保留所有现有功能。

**当前状态:** 使用 Commander.js 的单文件 CLI (`src/cli/index.ts`, 690 行)，需要记忆 8 个类别的 24 个命令。

**目标状态:** 使用 **Ink** (React for CLI) 的交互式 TUI，具有分层菜单、键盘导航、表单输入和实时反馈。

**推荐框架:** **Ink** (基于 React 的 TUI)
- ✅ 35.5k GitHub stars，积极维护 (2026 年 3 月)
- ✅ 原生 TypeScript 支持
- ✅ 被 Claude Code、GitHub Copilot CLI、Cloudflare Wrangler 使用
- ✅ 完美匹配 RSS2Pod 的 TypeScript + ESM 技术栈
- ✅ 优秀的测试支持 (React Testing Library)

## 1. 架构概览

### 1.1 TUI 框架选择：Ink

**选择 Ink 的理由:**
- **TypeScript 原生**: 用 TypeScript 编写，完美匹配 RSS2Pod 技术栈
- **React 范式**: 基于组件的架构，现代开发者熟悉
- **积极维护**: 2026 年 3 月更新，Vercel 支持
- **大规模验证**: 被生产级 CLI 使用 (Claude Code, GitHub Copilot CLI)
- **测试**: 完整的 React Testing Library 支持

## 7. 从 CLI 到 TUI 的迁移策略

### 阶段 1: 并行开发 (第 1-3 周)

**策略**: 在不修改 CLI 的情况下构建 TUI

```
第 1 周:
- 设置 Ink 框架
- 构建核心组件
- 创建状态管理

第 2 周:
- 构建菜单系统
- 创建类别界面

第 3 周:
- 在 TUI 中实现所有 24 个命令
- 提取共享处理器
```

**关键决策**: 尽早将命令逻辑提取到共享处理器，避免重复

### 阶段 2: 集成 (第 4 周)

**策略**: 统一 CLI 和 TUI 使用相同的命令处理器

```typescript
// 之前：CLI 有内联逻辑
program
  .command('status')
  .action(async () => {
    // 内联逻辑
  });

// 之后：CLI 使用共享处理器
import { showSystemStatus } from '../tui/commands/system.js';

program
  .command('status')
  .action(async () => {
    await showSystemStatus(); // TUI 共用
  });
```

**变更**:
1. 将所有命令逻辑提取到 `src/tui/commands/`
2. CLI 导入并调用共享处理器
3. TUI 导入并调用相同的处理器
4. 添加 `--tui` 标志启动 TUI: `npm run cli -- --tui`

### 阶段 3: 过渡 (第 5 周+)

**策略**: 使 TUI 成为默认，保持 CLI 可访问

```bash
# 默认启动 TUI
npm run cli
# → 打开交互式 TUI

# 传统 CLI 仍然工作
npm run cli -- status
npm run cli -- group:list -e

# 显式启动 TUI
npm run tui
```

**文档更新**:
- 更新 README 以 TUI 为特色
- 添加 TUI 快速入门指南
- 包含键盘快捷键速查表
- 记录 CLI 弃用时间表
