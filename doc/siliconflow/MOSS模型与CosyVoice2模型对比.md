# MOSS 模型与 CosyVoice2 模型功能差异对比

本文档详细对比 SiliconFlow 平台上 MOSS 模型 (fnlp/MOSS-TTSD-v0.5) 和 CosyVoice2 模型 (FunAudioLLM/CosyVoice2-0.5B) 的功能差异。

---

## 一、模型基本信息

| 属性 | MOSS 模型 | CosyVoice2 模型 |
|------|-----------|------------------|
| 模型名称 | fnlp/MOSS-TTSD-v0.5 | FunAudioLLM/CosyVoice2-0.5B |
| 模型类型 | 开源双语语音合成模型 | 多语言语音合成模型 |
| 官方定位 | 文本转语音对话 (Text to Spoken Dialogue) | 多语言情感语音合成 |

## 三、核心功能对比

### 3.1 MOSS 模型独有功能

| 功能 | 说明 |
|------|------|
| **双人对话合成** | 支持使用 `[S1]` 和 `[S2]` 标签区分两个说话人，自动生成双人对话音频 |
| **长篇语音生成** | 针对长文本优化，低延迟稳定输出，适合 AI 播客制作 |
| **max_tokens 控制** | 支持 max_tokens 参数控制生成 token 数量 |

### 3.2 CosyVoice2 模型独有功能

| 功能 | 说明 |
|------|------|
| **跨语言语音合成** | 支持不同语言之间的语音转换 |
| **情感控制** | 支持快乐、兴奋、悲伤、愤怒等多种情感表达 |
| **细粒度控制** | 通过富文本或自然语言对语音的情感和韵律进行精细控制 |
| **方言支持** | 支持中文方言：粤语、四川话、上海话、郑州话、长沙话、天津话 |
| **副语言标记** | 支持 `[laughter]`（笑声）、`[breath]`（呼吸）等标记 |
| **用户预置音色** | 支持通过 base64 或文件上传自定义音色 |
| **用户动态音色** | 支持动态音色管理（上传、列表、删除） |

### 3.3 两者都支持的功能

| 功能 | MOSS 模型 | CosyVoice2 模型 |
|------|-----------|------------------|
| 中英双语支持 | ✅ | ✅ |
| 声音克隆 | ✅ | ✅ |
| 预置音色（8种） | ✅ | ✅ |
| 流式传输 | ✅ | ✅ |
| 音频速度控制 | ✅ (0.25-4.0) | ✅ (0.25-4.0) |
| 音频增益控制 | ✅ (-10 到 10) | ✅ (-10 到 10) |
| 输出格式 | mp3/opus/wav/pcm | mp3/opus/wav/pcm |
| 采样率控制 | ✅ | ✅ |

## 四、输入方式对比

### 4.1 MOSS 模型输入格式

```
[S1]Hello, how are you today?[S2]I'm doing great, thanks for asking![S1]That's wonderful to hear.
```

- 使用 `[S1]` 和 `[S2]` 标签标识说话人
- 适合双人对话脚本
- 不支持 `<|endofprompt|>` 分隔符

### 4.2 CosyVoice2 模型输入格式

```
Can you say it with a happy emotion? <|endofprompt|>I'm so happy, Spring Festival is coming!
```

- 使用 `<|endofprompt|>` 分隔符区分指令和内容
- 支持自然语言情感描述
- 支持富文本标记（[laughter]、[breath]等）

## 五、API 参数对比

| 参数 | MOSS 模型 | CosyVoice2 模型 | 说明 |
|------|-----------|------------------|------|
| model | fnlp/MOSS-TTSD-v0.5 | FunAudioLLM/CosyVoice2-0.5B | 模型标识 |
| input | 必填 | 必填（带默认值） | 待转换文本 |
| voice | 可选 | 可选 | 预置音色 |
| references | 可选（双人） | 可选（克隆） | 参考音频 |
| max_tokens | ✅ 支持（默认2048） | ❌ 不支持 | 生成最大token数 |
| stream | ✅ 默认 true | ✅ 支持 | 流式传输 |
| speed | ✅ 默认 1.0 | ✅ 默认 1.0 | 语速 0.25-4.0 |
| gain | ✅ 默认 0 | ✅ 默认 0 | 增益 -10 到 10 |
| response_format | ✅ mp3/opus/wav/pcm | ✅ mp3/opus/wav/pcm | 输出格式 |
| sample_rate | ✅ 支持 | ✅ 支持 | 采样率控制 |

## 六、适用场景推荐

| 场景 | 推荐模型 | 原因 |
|------|----------|------|
| AI 播客/双人对话 | **MOSS** | 唯一支持双人对话合成的模型 |
| 长文本语音生成 | **MOSS** | 低延迟稳定输出 |
| 多语言内容 | **CosyVoice2** | 支持日、韩等多种语言 |
| 方言内容 | **CosyVoice2** | 支持多种中文方言 |
| 情感表达语音 | **CosyVoice2** | 支持多种情感控制 |
| 细粒度控制需求 | **CosyVoice2** | 支持自然语言指令和富文本标记 |
| 声音克隆 | 两者都支持 | 都能通过 references 实现 |

## 八、总结

### MOSS 模型优势
- ✅ 唯一支持双人对话合成
- ✅ 长文本处理更稳定
- ✅ 适合 AI 播客制作

### CosyVoice2 模型优势
- ✅ 多语言支持（中日韩）
- ✅ 中文方言支持
- ✅ 情感控制能力强
- ✅ 细粒度控制灵活
- ✅ 用户自定义音色功能完善
